# 卡帕西知识库构建规则 (Claude.md)

## 一、知识库目标

基于提供的化学/化工教材内容，构建一个**可增长的多课程化学知识库**，最终覆盖：
- 化工流程（已完成 4 章）
- 化工原理（待建设）
- 有机化学（规划中）
- 未来可扩展更多课程

用户可通过前端界面：
- 浏览知识条目（类似百科）
- 对不明白的知识点点击获取详细解释或进行 AI 问答
- 完成配套习题并获得讲解（解答默认折叠）
- 提交人工反馈以持续改进知识库内容

### 项目结构

```
├── docs/                    ← VitePress 站点内容（按课程分目录）
│   ├── .vitepress/          ← 站点配置
│   ├── index.md             ← 知识库总首页（课程选择器）
│   ├── 化工流程/
│   ├── 化工原理/
│   └── 有机化学/
├── raw/                     ← 原始资料（按课程隔离）
│   ├── <课程名>/ebook/      ← PDF，不进 git
│   ├── <课程名>/scan/       ← 扫描件，不进 git
│   └── <课程名>/text/       ← OCR 文本，进 git 追踪
├── scripts/                 ← Python 工具（按课程隔离）
└── .github/workflows/       ← CI/CD 自动部署
```

## 二、原始资料处理规则
### 2.1 来源格式
原始资料为 PDF 扫描件经 OCR 后的文本，可能包含：
- 正文段落
- 数学公式（LaTeX 或纯文本）
- Python 代码块
- 表格、示意图描述
- 分页不连续

### 2.2 预处理要求
1. **忽略页码与间断**  
   当文本出现明显的页码标记、页眉页脚或重复的标题时，按语义合并为连续叙述。
2. **代码块处理**  
   - 识别 `import`、`def`、`for` 等关键词的 Python 代码段，默认**剥离为独立附件**（如代码示例），不在百科正文中直接展示大段代码。
   - 若代码用于说明计算逻辑，可将其转换为伪代码或自然语言步骤描述。
3. **公式统一**  
   将所有数学表达式转换为 LaTeX 格式，确保前端可渲染。
4. **表格数据**  
   保留为 Markdown 表格，必要时补充表头与单位。
5. **图表与示意图**  
   原始扫描中若无矢量图，仅保留文字描述，并在相应位置插入占位标记 `[示意图：xxx]`。

## 三、百科文章生成规则
### 3.1 组织结构
知识库按**课程-章-节-条目**四级层次组织：
- **课程**：对应一门课程（如”化工流程”、”化工原理”），在 `docs/<课程名>/` 下
- **章**：对应教材的章（如”相平衡基础”、”换热器设计”），在课程目录下的子文件夹
- **节**：对应教材的次级标题（如”气液相平衡”、”对流传热系数”）
- **条目**：最小知识单元，每个概念、公式、方法或例题生成一个独立页面

所有课程共享同一 VitePress 实例和 KaTeX 渲染引擎。

### 3.2 内容格式
每篇百科文章包含：
- **标题**：概念或方法名称
- **定义/概述**（1~2 句话）
- **详细说明**（包含推导、适用条件、工程意义）
- **相关公式**（LaTeX 渲染）
- **关键参数**（符号、单位、含义）
- **计算示例**（如有例题，摘录并逐步讲解）
- **关联条目**（内部链接）
- **代码实现（可选）**：以折叠块形式附加，默认不展开

### 3.3 生成顺序
优先为以下内容创建条目：
1. 基本概念（如相平衡常数、泡点/露点、闪蒸、吸收因子）
2. 重要方程（如 Antoine 方程、Peng-Robinson 状态方程、Fenske 方程）
3. 工程方法（如 FUG 法、平均吸收因子法）
4. 典型例题（每道习题生成独立讲解页）

## 四、习题处理规则
1. **习题抓取**  
   从“习题”章节中提取题目，按编号归类（如 [4.1]、[5.3]）。
2. **配套讲解页**  
   为每道题生成单独页面，包含：
   - 题目原文（含数据）
   - 已知条件整理（列表）
   - 求解思路（分步自然语言）
   - 详细计算过程（手算步骤 + 若原书有程序可附简化伪代码）
   - 最终答案
3. **习题索引**  
   按章生成习题列表，可跳转到讲解页。

## 五、前端与交互功能设计

### 5.1 技术选型
- **静态站点框架**：VitePress（轻量级，Markdown 原生支持，Vue 组件可嵌入）
- **数学渲染**：KaTeX（通过 markdown-it-katex 插件，SSR + 客户端双渲染）
- **搜索**：VitePress 内置本地搜索
- **部署**：GitHub Pages + GitHub Actions 自动构建

### 5.2 本地开发命令

```bash
npm run dev      # 启动开发服务器 → http://localhost:5173
npm run build    # 构建静态站点 → docs/.vitepress/dist/
npm run preview  # 预览构建结果
```

### 5.2 知识浏览
- 左侧树形目录（章-节-条目）
- 右侧文章区，支持 LaTeX 数学公式、Markdown 表格
- 内部链接：点击文中关键词跳转到对应条目

### 5.3 详细解释与 AI 问答
- 每个条目下方提供 **“深入解释”** 按钮，点击后弹出对话框，由 AI 根据当前条目内容进行扩展讲解
- **AI 问答模块**：用户输入自然语言问题，系统检索知识库并调用大语言模型生成回答，回答中引用知识库具体条目
- 实现方式：前端请求后端 API，后端用向量数据库（如 Chroma/Pinecone）存储条目 Embedding，结合 LLM（如 GPT-4、Claude）进行 RAG 回答

### 5.4 人工反馈
- 每个条目页末尾设置 **“提交反馈”** 表单，支持：
  - 指出错误（勾选错误类型：公式错误、描述不准、缺失内容）
  - 提交修正建议（文本框）
  - 评分（⭐⭐⭐⭐⭐）
- 反馈存入后台数据库，管理员可查看并审核，更新知识库。

### 5.5 习题区
- 独立“习题集”页面，按章节筛选
- 每道题可点击“显示解答”，默认隐藏解答过程，鼓励先自行练习
- 解答页底同样提供 AI 对话入口：“对这道题有疑问？”

## 六、部署与持续集成

### 6.1 自动部署
推送 `main` 分支后，GitHub Actions 自动执行：
1. 安装依赖 `npm ci`
2. 构建站点 `npm run build`
3. 部署到 GitHub Pages

### 6.2 添加新课程的完整流程

以"化工原理"为例：

```bash
# 1. 放 PDF
cp 化工原理.pdf raw/化工原理/ebook/

# 2. 复制脚本模板
cp -r scripts/化工流程/* scripts/化工原理/

# 3. 修改脚本中的路径指向新课程

# 4. 运行 OCR
python scripts/化工原理/ocr_process.py

# 5. 生成百科条目
python scripts/化工原理/generate_wiki.py

# 6. 生成习题解答
python scripts/化工原理/gen_solve_chX.py

# 7. 加折叠块
python scripts/化工原理/add_exercise_details.py

# 8. 注册导航 — 在 docs/.vitepress/config.mjs 添加 sidebar 和 nav 条目

# 9. 本地预览
npm run dev

# 10. 推送
git add . && git commit -m "新增化工原理" && git push
```

### 6.3 Git 管理规则
- **追踪**：`docs/`、`scripts/`、`raw/*/text/`、`package.json`、`.github/`
- **不追踪**：`raw/*/ebook/`、`raw/*/scan/`、`node_modules/`、`dist/`

## 七、数据维护与持续优化

### 7.1 版本管理与 Changelog 维护

**版本管理**：知识库内容使用 Git 仓库管理，所有文章为 Markdown 文件。

**Changelog 维护规范**：

- **更新频率**：每次重要改动后立即更新（不要等到月底）；使用 `/update-changelog` skill 快速记录
- **格式规范**：
  - 日期分组：`## YYYY-MM-DD`
  - 类别分组：`### 类别名称`
  - 内容形式：
    - 列表项：`- **粗体标题**：描述`
    - 表格行：用于多文件、多项改动的汇总
- **标准类别列表**：
  - 内容补全
  - 公式修复
  - 兼容性清理
  - 知识库结构
  - 部署与基础设施
  - 网站功能
  - 其他
- **提交规范**：
  - 提交信息格式：`docs: 更新 CHANGELOG — YYYY-MM-DD 改动描述`
  - 每次 CHANGELOG 更新应单独提交或与相关改动一起提交

### 7.2 AI 辅助生成流水线

- 将预处理后的文本提交给 Claude，按本章规则要求生成初步条目草稿
- 人工校对后入库

### 7.3 反馈驱动更新

分析高频反馈，优先修正错误或补充需求强烈的知识点。

## 八、本文件用途
`Claude.md` 作为系统规则文件，用于指导所有 AI（Claude）生成知识库内容时的行为。  
- 提供给大模型作为 system prompt 的一部分
- 开发人员读取此文件了解项目规范

后续所有知识库内容创建，均应遵循本规则。
